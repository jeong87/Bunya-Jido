# Repository Thesis - Bunya-Jido

## One-sentence thesis

Bunya-Jido publishes evidence-linked repository interpretations as offline
maps and bounded coding-agent context only after explicit trust gates pass.

## Audience and onboarding question

A newcomer should first understand how authored semantic claims travel through
validation into a human-readable map and a coding-agent route contract.

## Evidence supporting this thesis

- `src/bunya_jido/cli.py` exposes build, validation, diagnostic, context, and
  atlas-quality commands.
- `src/bunya_jido/blueprint.py` validates and publishes semantic graph
  artifacts and agent-map routes.
- `src/bunya_jido/quality.py` evaluates measurable Studio readability and
  scenario-policy signals without claiming editorial proof.
- `src/bunya_jido/viewer/index.template.html` renders trust, projections, and
  validated scenario playback offline.
- `tests/test_self_map.py` and `tests/test_studio_benchmark.py` protect the
  published self-map and cross-domain contract.

## What this repository is not best explained as

It is not merely a directory scanner, a decorative graph renderer, or a
domain-specific workflow animator. Static observations support publication;
they do not replace reviewed semantic claims.

## Ordered behavior assessment

- Strength: strong
- Scenario policy recommendation: required
- Risks of over-narration: Treating an illustrative projection as execution,
  or presenting an unvalidated route as trusted agent context.
