# Projection Candidates - Bunya-Jido

## Candidate A: Trusted Publication

- **Question answered:** How does an authored interpretation become a map that
  people and coding agents may trust?
- **Landmark nodes:** Command Line Interface, Blueprint Projection, Grounding
  Gate, Atlas Quality Evaluator, HTML Renderer, Interactive Viewer.
- **Possible planes and project-local vocabulary:** entry, semantic, quality,
  presentation; commands, projections, gates, evidence, surfaces.
- **Grounding strength:** Strong; each boundary has source or test evidence.
- **First-screen value:** Shows the publication boundary before any playback.
- **Distortion risks:** Hiding the agent-route contract as a secondary detail.
- **Scenario potential:** Strong documented behavioral path.

## Candidate B: Bounded Agent Navigation

- **Question answered:** How does a coding task obtain trusted reading and test guidance?
- **Landmark nodes:** Command Line Interface, Trusted Context Generator, Agent
  Route Parity, Grounding Gate, Semantic Contract Tests.
- **Grounding strength:** Strong for context and validation commands.
- **First-screen value:** Useful for agent adopters, less complete for humans
  first discovering the map product.
- **Distortion risks:** Making the viewer appear secondary to agent routing.
- **Scenario potential:** Strong documented behavioral path.

## Candidate C: Quality And Benchmark Contract

- **Question answered:** How are readability and cross-domain honesty checked?
- **Landmark nodes:** Atlas Quality Evaluator, Studio Benchmark Suite, Semantic
  Contract Tests, Continuous Integration.
- **Grounding strength:** Strong but supporting rather than the product entry.
- **Scenario potential:** Better as a secondary projection than a primary story.

## Editorial selection

- **Primary projection:** Trusted Publication
- **Why it wins:** It explains the common human and coding-agent artifact while
  keeping trust visible before scenario playback.
- **Secondary views to retain:** Bounded Agent Navigation and Quality Contract.
- **Components that must not be over-centralized:** Static Scanner and CI are
  support evidence, not the authored meaning of a map.
- **Static overlays permitted in the primary view, if any:** None.
